# Agent 工程化总体学习文档

## 适用画像
- 有后端工程经验
- 熟练使用 Agent 编程工具
- 目标是落地生产级 Agent 系统，而非入门 Demo

## 8周进阶学习目标
- 交付一个可上线的 Agent 系统：可观测、可回归、可灰度、可降级。
- 建立工程标准：评测、SLO、成本控制、权限边界、变更流程。

## 8周学习计划（总览）
### W1：架构基线
- 任务：明确单Agent/多Agent边界、上下文模型、状态存储、工具契约（JSON Schema）
- 产物：`architecture.md`、`contracts/`、主流程时序图

### W2：Workflow 内核
- 任务：实现 Planner-Executor-Replanner（含重试、超时、幂等）
- 产物：可恢复执行引擎（失败可续跑）

### W3：RAG 工程化
- 任务：分层检索（召回+重排+压缩）、索引版本化、缓存策略
- 产物：可回放检索链路、离线评测集 v1

### W4：工具治理
- 任务：工具注册中心、权限模型、参数校验、熔断与降级
- 产物：工具调用网关、审计日志

### W5：评测体系
- 任务：成功率/延迟/成本四维评测、回归基线
- 产物：`eval/` 自动化评测脚本、CI 门禁

### W6：可观测性
- 任务：全链路 Trace、错误分桶、慢链路定位
- 产物：仪表盘、TopN 失败案例报告

### W7：多Agent协作
- 任务：角色分工（规划/执行/审计）、冲突仲裁、共享记忆一致性
- 产物：多Agent业务流程 Demo（>=2个场景）

### W8：上线与治理
- 任务：灰度发布、A/B、回滚预案、SLA/SLO、Runbook
- 产物：生产发布包、运维手册、复盘报告

## 每周固定验收指标
- 任务端到端成功率（E2E）
- P95 延迟
- 单任务成本（Token + 工具耗时）
- 工具错误率与重试成功率
- 回归集通过率（不低于上周基线）

## 实施路线（可选）
- 方案A（推荐）：先单Agent做深（稳定性/评测/治理），再扩展多Agent
- 方案B：先多Agent协作框架，再补治理与评测
