# W1 执行清单（按天拆解，后端工程化版）

## W1目标（对应你8周计划的第1周）
- 明确单Agent/多Agent边界与上下文流转模型。
- 定义状态存储与恢复策略（幂等、重试、超时、续跑）。
- 完成工具契约（JSON Schema）与调用网关基础约束。
- 产出可评审架构文档与最小可运行骨架。

## Day 1（边界与场景基线）
- 任务
  - 定义3个核心业务场景（高频、复杂、失败代价高各1个）。
  - 画出单Agent与多Agent责任边界（谁决策、谁执行、谁审计）。
  - 输出关键非功能目标：SLO、成本上限、可观测最小集。
- 输出物
  - `docs/architecture/boundaries.md`
  - `docs/adr/0001-agent-boundary.md`
- 验收标准
  - 每个场景都有输入、输出、失败路径、人工接管点。

## Day 2（上下文模型设计）
- 任务
  - 定义上下文分层：系统上下文、任务上下文、工具上下文、会话记忆。
  - 设计上下文生命周期：创建、裁剪、持久化、过期、清理。
  - 明确敏感字段脱敏规则与日志红线字段。
- 输出物
  - `docs/architecture/context-model.md`
  - `src/agent/context.py`（数据结构骨架）
- 验收标准
  - 任一请求都可追踪“上下文来源”和“裁剪策略”。

## Day 3（状态机与恢复）
- 任务
  - 设计状态机：`PENDING -> RUNNING -> WAITING_TOOL -> SUCCESS/FAILED`。
  - 定义可重试错误码与不可重试错误码。
  - 设计快照点与恢复入口（按 step_id/session_id）。
- 输出物
  - `docs/architecture/state-machine.md`
  - `src/state/store.py`
  - `src/state/recovery.py`
- 验收标准
  - 失败后可从最近快照续跑，不重复执行已完成步骤。

## Day 4（工具契约与治理）
- 任务
  - 为核心工具定义 JSON Schema（参数、必填、枚举、边界）。
  - 定义工具级权限（谁可调用、何种上下文可调用）。
  - 约定统一返回结构：`success/data/error/retryable`。
- 输出物
  - `contracts/tools/*.schema.json`
  - `src/gateway/validator.py`
  - `docs/adr/0003-tool-contract-governance.md`
- 验收标准
  - 契约校验失败时，能返回明确错误并阻断执行。

## Day 5（主流程时序与最小骨架）
- 任务
  - 产出主链路时序图：请求 -> 规划 -> 工具 -> 汇总 -> 响应。
  - 打通最小可运行流程（mock工具即可）。
  - 接入审计日志（请求ID、stepID、toolID、耗时、结果）。
- 输出物
  - `docs/architecture/sequence-main-flow.md`
  - `src/agent/planner.py`
  - `src/agent/executor.py`
  - `src/gateway/audit_logger.py`
- 验收标准
  - 本地可完成1条端到端请求，日志可完整回放。

## Day 6（契约测试与集成测试）
- 任务
  - 写契约测试：合法参数、缺参、类型错、越界值。
  - 写集成测试：工具超时、工具失败、恢复重放。
  - 增加脚本：批量校验 schema + 本地回放会话。
- 输出物
  - `tests/contract/*`
  - `tests/integration/*`
  - `scripts/validate_contracts.sh`
  - `scripts/replay_session.sh`
- 验收标准
  - 契约测试通过率100%，关键失败路径均有测试覆盖。

## Day 7（评审与冻结）
- 任务
  - 组织一次架构评审（重点：边界、状态恢复、契约治理）。
  - 汇总风险清单（技术债、稳定性、安全、成本）。
  - 冻结W1基线，输出W2实施输入（Planner-Executor-Replanner细化）。
- 输出物
  - `docs/architecture/review-notes-w1.md`
  - `docs/adr/0002-state-store-selection.md`
  - `README.md`（启动、测试、回放说明）
- 验收标准
  - 评审结论明确：保留项/整改项/延期项；W2无阻塞输入。

## W1完成定义（DoD）
- 架构文档完整：边界、上下文、状态机、时序图齐全。
- 契约落地：核心工具 schema + 校验器已接入。
- 可运行：至少1条端到端主流程可执行并可回放。
- 可测试：契约测试与关键失败路径集成测试可运行。
- 可评审：ADR与评审记录可追溯。

## 可选替代方案（按团队情况二选一）
- 方案A（稳态优先，推荐）：先收敛单Agent边界与契约，再上多Agent。
- 方案B（扩展优先）：先并行设计多Agent协议，再补契约治理与恢复。
