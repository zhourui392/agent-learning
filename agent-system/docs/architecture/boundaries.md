# 智能体边界基线

## 范围
本文定义了面向生产的单智能体与多智能体职责基线，并给出 W2 Workflow 内核阶段的职责划分。

## 核心场景
### 场景 A：高频场景 - 客户常见问题（FAQ）检索
- 输入：用户问题、locale、tenant_id。
- 输出：带来源片段的简洁回答。
- 失败路径：检索超时或召回为空。
- 人工接管：当置信度 < 0.6 时转入客服队列。

### 场景 B：高复杂场景 - 多步骤数据分析
- 输入：分析需求、指标定义、时间范围。
- 输出：聚合结果、前置假设与置信度。
- 失败路径：中间工具失败、依赖阻塞或数据不一致。
- 人工接管：由评审者校验生成 SQL 与分析摘要。

### 场景 C：高失败成本场景 - 通知自动化
- 输入：通知意图、接收人、approval_token。
- 输出：发送状态与 audit_id。
- 失败路径：权限拒绝、重复发送、通道故障。
- 人工接管：人工审批并决定是否重发。

## 职责边界
### 单智能体（默认）
- Planner：负责意图解析、计划生成、风险标记、计划图校验。
- Executor：负责步骤调度（串并混合）、超时/重试/取消控制、状态推进。
- Replanner：负责失败后计划调整、步骤映射、策略决策。
- Gateway + Audit：负责契约校验、工具调用治理、审计日志。

### 多智能体（未来）
- 规划智能体（Planner Agent）：拆解任务并构建依赖图。
- 执行智能体（Executor Agent）：执行工具调用并上报结构化结果。
- 审计智能体（Auditor Agent）：校验策略、Schema 与风险阈值。
- 协调器（Coordinator）：处理冲突并合并最终响应。

## 决策 / 执行 / 审计 分离
- 决策责任人：planner + replanner 角色。
- 执行责任人：executor 角色。
- 审计责任人：gateway + audit logger（审计日志器）。

## 非功能性目标（W2 基线）
- 服务级目标（SLO）：
  - 在冒烟用例集上端到端成功率 >= 95%。
  - 单工具流程 P95 延迟 <= 8s。
- 成本上限：
  - 平均每请求工具调用次数 <= 3。
  - 平均每请求 token 预算 <= 8k。
- 最低可观测性要求：
  - 必填字段：request_id、session_id、trace_id、plan_version、step_id、tool_id、duration_ms、result_status。

## 风险控制
- 拒绝所有未通过 Schema 校验的工具调用。
- 不可重试错误立即停止并输出明确错误码。
- 写操作必须显式携带幂等键。
- 重规划必须保留 step_id 映射和历史轨迹。
