# 幂等与续跑（W2）

## 幂等键规则
- 幂等键：`{request_id}:{step_id}:{plan_version}`。
- 作用域：步骤级。
- 写入时机：步骤成功后落库。

## 幂等判定
- 若步骤状态为 `SUCCESS` 且幂等键匹配，则跳过执行并复用历史结果。
- 若步骤失败或幂等键不匹配，则进入正常执行流程。

## 快照点
- `pre-step`：步骤开始前。
- `post-success`：步骤成功后。
- `final-response`：响应构建前。

## 断点续跑
- 入口参数：`session_id` + 可选 `step_id`。
- 指定 `step_id`：从该步骤最近快照恢复。
- 不指定 `step_id`：从最近成功快照后的首个未完成步骤恢复。

## 失败恢复约束
- 不重复执行已成功且幂等键一致的步骤。
- 失败步骤允许按重试/重规划策略继续。
- 恢复过程必须保留历史审计与快照链路。
