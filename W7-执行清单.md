# W7 执行清单（按天拆解，多 Agent 协作）

## W7目标（对应8周计划第7周）
- 完成多角色协作流程（Planner/Executor/Auditor）。
- 解决协作冲突、状态一致性和责任边界问题。
- 产出可演示的多 Agent 业务流程。

## Day 1（角色建模）
- 任务
  - 定义角色职责边界与输入输出。
  - 约定角色间 SLA（响应时限、错误处理责任）。
  - 明确人工介入触发点。
- 输出物
  - `docs/multi-agent/roles-and-responsibilities.md`
  - `docs/multi-agent/sla.md`
- 验收标准
  - 每个角色职责单一且无冲突重叠。

## Day 2（消息协议）
- 任务
  - 定义消息协议（header、payload、meta、version）。
  - 定义状态字段与冲突字段。
  - 实现协议校验器。
- 输出物
  - `contracts/multi-agent-message.schema.json`
  - `src/multi_agent/protocol_validator.py`
- 验收标准
  - 跨角色消息可自动校验并兼容版本升级。

## Day 3（任务分派）
- 任务
  - 实现任务分派与回传通道。
  - 增加超时回收与重试策略。
  - 记录角色级审计日志。
- 输出物
  - `src/multi_agent/dispatcher.py`
  - `src/multi_agent/callback_handler.py`
- 验收标准
  - 任务可稳定分发并收敛到统一结果。

## Day 4（冲突仲裁）
- 任务
  - 定义冲突类型（结果冲突、证据冲突、优先级冲突）。
  - 实现仲裁规则（规则优先、置信度、人工兜底）。
  - 保留仲裁过程证据链。
- 输出物
  - `src/multi_agent/arbitrator.py`
  - `docs/multi-agent/arbitration-policy.md`
- 验收标准
  - 冲突场景有稳定可重复的决策结果。

## Day 5（共享记忆一致性）
- 任务
  - 设计共享记忆结构与版本控制。
  - 处理并发写冲突（乐观锁/版本号）。
  - 增加过期与清理策略。
- 输出物
  - `src/multi_agent/shared_memory.py`
  - `docs/multi-agent/memory-consistency.md`
- 验收标准
  - 并发写入下记忆状态一致、可追踪。

## Day 6（业务 Demo）
- 任务
  - 打通至少2个多Agent业务流程（标准流程 + 异常流程）。
  - 录制回放数据用于评测。
  - 对比单Agent与多Agent效果。
- 输出物
  - `demo/multi_agent_case_1.md`
  - `demo/multi_agent_case_2.md`
  - `eval/reports/w7-single-vs-multi.md`
- 验收标准
  - Demo 可稳定复现，效果对比结论明确。

## Day 7（评审与冻结）
- 任务
  - 评审多Agent架构复杂度与收益。
  - 冻结角色协议和协作边界。
  - 输出 W8 上线准备输入。
- 输出物
  - `docs/review/review-notes-w7.md`
  - `docs/handover/w7-to-w8.md`
- 验收标准
  - W8 上线工作无协议阻塞。

## W7完成定义（DoD）
- 多角色流程可运行且可回放。
- 协议稳定：版本、校验、兼容策略齐全。
- 冲突可处理：自动仲裁 + 人工兜底可用。
- 一致性可保证：共享记忆并发写入可控。

## 可选替代方案
- 方案A（推荐）：先两角色协作，再扩展审计角色。
- 方案B：一次性三角色并行，后续再做协议收敛。
