# Incident Handling

## 适用范围
- 发布期异常、实验异常、核心依赖抖动、人工接管场景。

## 标准步骤
1. 确认影响范围：用户数、租户、区域、开始时间。
2. 确认当前档位：灰度比例、实验开关、最近一次操作人。
3. 检查核心指标：`success_rate`、`latency.p95_ms`、`timeout_rate`。
4. 选择动作：冻结放量、回滚、限流、降级、人工接管。
5. 验证恢复：白名单冒烟、监控回稳、告警收敛。

## 常见故障
| 故障 | 首选动作 | 兜底动作 |
|------|----------|----------|
| 成功率骤降 | 关闭实验 + 回滚 | 人工接管 |
| 延迟持续升高 | 冻结放量 + 限流 | 回滚 |
| 外部依赖超时 | 熔断 + 降级 | 人工接管 |
| 仲裁结果异常 | 切回单 Agent / control | 人工复核 |

## 结束条件
- 指标恢复到基线区间并稳定 30 分钟。
- 事故单已补齐根因、处置动作和后续责任人。
