# On-call Escalation

## 值班链路
- 一线：当班 On-call，5 分钟内响应。
- 二线：模块负责人，15 分钟内接管。
- 三线：系统 owner / 应急群，30 分钟内给出决策。

## 标准流程
1. 接收告警并确认影响范围。
2. 打开 Dashboard 查看成功率、P95、错误分桶。
3. 按 `trace_id` 查看 `logs.jsonl`、`traces.jsonl`、失败样本归档。
4. 根据 Runbook 执行降级、回滚、限流或人工接管。
5. 记录处理结论与后续动作。

## 升级条件
- 连续 15 分钟无缓解。
- 影响核心链路或多租户。
- 存在数据错误或安全风险。
