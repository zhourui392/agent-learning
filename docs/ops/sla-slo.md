# SLA / SLO

## 对外 SLA
| 指标 | 目标 |
|------|------|
| 可用性 | `99.5%` / 月 |
| P95 响应时间 | `<= 3s` |
| P99 响应时间 | `<= 5s` |
| P1 响应时效 | 5 分钟内确认 |

## 内部 SLO
| 指标 | 目标 | 观察窗口 |
|------|------|----------|
| `success_rate` | `>= 99.0%` | 5 分钟 |
| `latency.p95_ms` | `<= 2500` | 5 分钟 |
| `timeout_rate` | `<= 1.0%` | 15 分钟 |
| `manual_takeover_rate` | `<= 0.5%` | 1 天 |

## 错误预算
- 月度不可用预算：`0.5%`。
- 单日消耗超过月预算的 `20%` 时，冻结一切放量与实验。
- 月预算耗尽后，只允许修复性发布。

## 运营要求
- 每个核心指标都要有 dashboard、告警规则与值班负责人。
- 每次发布前确认错误预算余额、最近 24 小时告警与未关闭事故。
