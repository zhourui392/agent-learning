# 降级策略（Degrade Policy）

## 概述

定义 RAG 检索链路在异常场景下的降级行为，确保即使检索服务部分不可用，系统仍能给出可接受的响应。

---

## 1. 降级触发条件

| 条件 | 阈值 | 说明 |
|------|------|------|
| 检索超时 | > 3000ms | 召回 + 重排 + 压缩总耗时超限 |
| 向量服务不可用 | 连接失败 / 5xx | 向量数据库异常 |
| 关键词服务不可用 | 连接失败 / 5xx | ES 异常 |
| 全路召回为空 | 0 结果 | 无任何匹配 |

---

## 2. 降级行为

### 2.1 超时降级

```
正常路径：向量召回 + 关键词召回 → 重排 → 压缩 → 返回
超时降级：检测到耗时 > 阈值
  → 优先返回 query 缓存结果（标记 degraded）
  → 无缓存时返回 context 缓存
  → 均无时返回空结果 + 降级标记
```

### 2.2 单路故障降级

```
向量服务故障 → 降级为纯关键词检索
关键词服务故障 → 降级为纯向量检索
两路均故障 → 走超时降级流程
```

### 2.3 空结果处理

```
召回结果为空 → 返回空上下文 + 提示 LLM "未找到相关信息"
```

---

## 3. 缓存策略

### 3.1 缓存层级

| 层级 | Key | Value | TTL | 容量 |
|------|-----|-------|-----|------|
| Query 缓存 | sha256(query+role) | RetrievalResult | 5 分钟 | 500 条 |
| Embedding 缓存 | chunk_id | 向量数组 | 1 小时 | 5000 条 |
| Context 缓存 | sha256(query+role) | 组装后文本 | 10 分钟 | 200 条 |

### 3.2 缓存失效

| 场景 | 策略 |
|------|------|
| TTL 到期 | 自动过期，下次查询 MISS 后重新填充 |
| 数据源更新 | 清空 query + context 缓存，按 source_id 清 embedding |
| 手动清理 | 管理接口支持按前缀 / 全量清理 |

### 3.3 缓存安全

- **缓存 key 包含用户角色**：防止低权限用户命中高权限缓存
- **缓存不存储原始密文**：仅缓存脱敏后的检索结果

---

## 4. 监控指标

| 指标 | 告警阈值 | 说明 |
|------|----------|------|
| 缓存命中率 | < 20% | 命中率过低说明缓存策略无效 |
| 降级触发率 | > 5% | 频繁降级说明后端服务不稳定 |
| 平均检索延迟 | > 2000ms | 接近超时阈值 |
| 空结果率 | > 10% | 检索质量问题 |
