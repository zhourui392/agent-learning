# 知识源清单（Knowledge Inventory）

## 概述

本文档梳理 RAG 系统所需对接的全部知识源，按数据特征分为三大类：静态文档、动态 API、结构化数据库。每类数据源均定义了同步频率、失效策略和索引粒度。

---

## 1. 知识源分类

### 1.1 静态文档（Static Documents）

| 数据源 | 格式 | 典型规模 | 索引粒度 | 同步频率 | 失效策略 |
|--------|------|----------|----------|----------|----------|
| 产品文档 | Markdown / PDF | 数百篇 | 段落级（paragraph） | 文件变更触发（git hook / webhook） | 文件删除时移除索引 |
| FAQ 知识库 | JSON / YAML | 数千条 | 条目级（entry） | 每日全量同步 | 标记废弃后 7 天删除 |
| 操作手册 | Markdown | 数十篇 | 段落级 | 文件变更触发 | 版本替换时标记旧版过期 |
| 合规与政策文件 | PDF / DOCX | 数十篇 | 段落级 | 手动发布触发 | 新版生效后旧版立即失效 |

### 1.2 动态 API（Dynamic API）

| 数据源 | 协议 | 典型 QPS | 索引粒度 | 同步频率 | 失效策略 |
|--------|------|----------|----------|----------|----------|
| 实时配置中心 | HTTP REST | 低（< 10） | 键值级（key-value） | 变更推送（长轮询 / WebSocket） | TTL 过期自动失效 |
| 外部搜索引擎 | HTTP REST | 中（< 100） | 文档级（document） | 实时调用，不缓存索引 | 无本地索引，每次实时查询 |
| 业务状态查询 | Dubbo RPC | 中 | 记录级（record） | 实时调用 + 本地缓存 60s | 缓存 TTL 到期后重新拉取 |

### 1.3 结构化数据库（Structured DB）

| 数据源 | 存储引擎 | 典型规模 | 索引粒度 | 同步频率 | 失效策略 |
|--------|----------|----------|----------|----------|----------|
| 业务主数据（商品/商户） | MySQL | 百万级行 | 行级（row） | CDC 增量同步（Debezium / Canal） | 逻辑删除标记后从索引移除 |
| 运营配置表 | MySQL | 千级行 | 行级 | 定时全量同步（每小时） | 配置变更后下一轮同步生效 |
| 日志与指标 | ElasticSearch | 亿级文档 | 文档级 | 实时写入 | 按保留期自动清理（默认 30 天） |

## 2. 索引粒度定义

| 粒度级别 | 说明 | 适用场景 | 典型 Token 数 |
|----------|------|----------|---------------|
| 文档级（document） | 整篇文档作为一个检索单元 | 短文档、FAQ 条目 | 200-2000 |
| 段落级（paragraph） | 按段落或语义段切分 | 长文档、技术手册 | 100-500 |
| 句级（sentence） | 按句子切分 | 精确事实检索 | 20-80 |
| 行级（row） | 数据库单行记录 | 结构化查询结果 | 50-200 |
| 键值级（key-value） | 单个配置项 | 配置/参数查询 | 10-50 |

### 粒度选择原则

1. **默认使用段落级**：平衡召回率与精度。
2. **FAQ / 短条目用文档级**：条目本身足够短，无需再切分。
3. **精确事实抽取用句级**：如数字、日期、名称等精确信息。
4. **结构化数据用行级**：保持数据完整性，配合 Schema 描述。

## 3. 数据更新与索引刷新策略

### 3.1 同步模式

| 模式 | 触发条件 | 延迟 | 适用数据源 |
|------|----------|------|------------|
| 事件驱动（push） | 文件变更 / DB CDC / 配置推送 | 秒级 | 静态文档、业务主数据、配置中心 |
| 定时拉取（pull） | 定时任务（cron） | 分钟~小时级 | FAQ 知识库、运营配置表 |
| 实时查询（pass-through） | 用户查询时实时调用 | 毫秒级 | 外部搜索引擎、业务状态查询 |

### 3.2 索引刷新流程

```
数据源变更
  → 变更检测（webhook / CDC / cron）
  → 文档预处理（清洗、切分、Embedding）
  → 写入向量存储 + 关键词索引
  → 更新版本号与时间戳
  → 旧版本标记过期
```

### 3.3 失效与清理

| 策略 | 说明 |
|------|------|
| TTL 过期 | 索引条目写入时附带 TTL，到期自动清理 |
| 版本替换 | 新版文档入库后旧版标记 `expired`，查询时过滤 |
| 逻辑删除同步 | 源端逻辑删除时，索引端同步标记并在下一轮清理 |
| 手动清理 | 管理接口支持按数据源 / 时间范围手动清理 |

## 4. 数据质量保障

### 4.1 入库前校验

- 文档非空检查：跳过空文件或全空白文件。
- 编码检查：统一转换为 UTF-8。
- 格式校验：Markdown 语法检查、JSON Schema 验证。
- 去重检查：基于内容哈希去重，避免重复索引。

### 4.2 索引健康监控

- 索引总量变化趋势：异常增减告警。
- 陈旧条目占比：超过阈值（如 20%）触发告警。
- Embedding 维度一致性：新入库向量维度必须与现有索引一致。
