# Pre-Prod Drill

## 演练目标
- 验证灰度、A/B、告警、回滚、人工接管的协同可用性。
- 在预发布环境完整走通一次上线前检查与异常处置流程。

## 演练范围
| 项目 | 目标 |
|------|------|
| 灰度放量 | 验证 `1% -> 5% -> 20%` 可按步骤执行 |
| A/B 实验 | 验证稳定分桶、白名单、手工覆盖、即时停止 |
| 告警联动 | 验证 `P1/P2` 触发与升级链路 |
| 回滚 | 验证 5 分钟内切回 `control` |
| 人工接管 | 验证接管、记录、恢复流程 |

## 演练前检查
1. 确认 `docs/release/gray-release-plan.md` 与 `docs/release/rollback-policy.md` 已评审。
2. 确认 dashboard、告警、值班链路已就绪。
3. 准备白名单账号、回放样本、故障注入脚本。

## 演练步骤
1. 从 `0%` 升到 `1%`，观察 30 分钟。
2. 开启 `A/B` 实验，验证白名单与手工覆盖。
3. 注入上游依赖超时，确认告警触发与值班升级。
4. 命中安全阈值后执行实验止损与版本回滚。
5. 验证人工接管与恢复自动链路。
6. 记录问题、责任人和修复截止时间。

## 通过标准
- 所有关键步骤都有明确 owner 和操作记录。
- 告警触发、回滚、人工接管均在预期时限内完成。
- 演练暴露问题全部进入 `docs/reports/w8-drill-issues.md` 闭环。
